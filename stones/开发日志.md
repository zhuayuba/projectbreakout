## 项目起点

**参考大纲：** Gemini 生成的第一期 4 周大纲（熵增、相变、反馈回路、激励与损失厌恶），但只取方向和大意，不完全照搬。

**首次交付：** 生成 `第一期_熵增与系统宿命.html`，暗色典雅风格，包含硬科学模型解说、明珠概念、晚间引子、对照映射、反向思考题。

## 功能迭代

### 第一版：共享 JS 库方案（失败）

**设计：** `芒格计划.js` 作为共享库，各期 HTML 通过 `<script src="...">` 加载。高亮通过选中文字自动弹出工具栏触发。

**问题：**
1. 中文文件名 `芒格计划.js` 在 `file://` 协议下加载失败
2. `mouseup` + `requestAnimationFrame` 的时序冲突——工具栏弹出后立刻被 `click` 事件关闭

**改成：** 重命名为 `munger.js`，改用 `selectionchange` 事件 + 80ms 防抖。

**依然失败：** 工具栏还是不弹出。

---

### 第二版：右键触发（重大突破）

**关键洞察：** 用户建议用右键代替自动弹出。右键是明确的用户意图，没有时序冲突。

**实现：** 监听 `contextmenu` 事件 → 缓存选中文本和 range → 弹出自定义工具栏 → 阻止浏览器默认右键菜单。

**结果：** 工具栏终于弹出来了！但标记数据不入库。

---

### 第三版：调试数据不入库的 bug（关键发现）

**现象：** 标记后 toast 提示正常，但侧边栏显示 0 条。诊断显示 `getIssueInfo → number=0 → 返回 null`。

**根因：** `.hero-badge` 里写的是"芒格计划 · **第**一**期**"，正则 `/\d+/` 只匹配阿拉伯数字，**"一"不是数字**。所以 `addHighlight` 因为 `info.number === 0` 直接 return null 了。

**修复：** 把 badge 改成"芒格计划 · **第1期**"，同时在 `info()` 函数里加了中文数字映射 `{"一":1,"二":2,...}`。

**教训：** 这个 bug 从第一天就在，因为之前工具栏一直弹不出来所以没暴露。

---

### 第四版：localStorage 不可用（关键发现）

**现象：** 标记后数据只在内存中，`localStorage` 写入失败。诊断显示 `localStorage ✗`。

**根因：** 浏览器对 `file://` 协议限制了 localStorage 写入。

**修复：** 数据同时存在内存数组 `_h` 和尝试写 localStorage。同页面内永远可用，即使 localStorage 写不进也不影响侧边栏。

---

### 第五版：ES6 语法在 file:// 下静默失败（终极发现）

**现象：** 好多次代码看起来正确，`node -c` 检查通过，但浏览器里完全没反应——没有浮动按钮，没有右键菜单。

**根因：** JS 里混用了 `const`、`let`、箭头函数、模板字符串。在 `file://` 协议下浏览器对 ES6 语法的解析行为不可预测，脚本在加载阶段就静默崩溃。

**修复：** 全部改用 `var` 和 `function`，全部字符串用双引号 `"`，禁止 `=>`，禁止 `` `${}` ``。包裹在 `(function(){"use strict"; ... })();` IIFE 中。

**结果：** 所有期页面终于全部正常工作。

---

### 第六版：笔记本从独立页面变成侧边栏

**用户需求：** 不要跳转到独立页面，在当前阅读页面就能查看笔记本。

**实现：** 点击右下角「📒 笔记本」按钮 → 右侧滑出 380px 宽侧边栏 → 半透明遮罩 → 按期次分组显示所有标记和笔记 → 支持编辑、删除 → 点击标记跳转到原文位置。

**好处：** 阅读和笔记无缝切换，数据在同一页面上下文中。

## 引文风波

**问题：** 用户发现 `"生命的本质，就是逆熵的存在。" —— 薛定谔` 不是薛定谔的原话。

**修正原则：**
1. 所有 `blockquote` 中的引文必须是可核实的原文
2. 中文翻译后标注准确的原文出处（作者、书名、年份、章节/页码）
3. 不允许自由发挥当成原文来引用
4. 如果实在找不到合适引文，宁可不用

**替换结果：**
- 第 1 期薛定谔引文 → 替换为《生命是什么》第 6 章真实引文
- 第 5 期丹尼特引文 → 替换为《达尔文的危险思想》第 21 页真实引文

## macOS 文件权限问题

**现象：** 旧的 `index.html` 和 `第一期.html` 无法通过 Read 工具、Edit 工具或 Bash 命令读取/编辑/删除（`Operation not permitted`）。

**根因：** macOS 的 `com.apple.provenance` 扩展属性锁定。

**解决：** 这些文件无法恢复。创建新文件代替——`芒格计划.html` 代替 `index.html`，`01.html` 代替 `第一期.html`。

## 经验汇总

1. **file:// 协议下只用 ES5**：`var` + `function`，不碰任何 ES6+ 语法。
2. **自包含文件**：不依赖外部 JS/CSS，全写在一个 HTML 里。
3. **右键触发 > 自动弹出**：确定性事件 > 时序敏感的检测。
4. **调试用逐步诊断**：在每个关键函数入口打印状态，逐环排查。
5. **正则要覆盖中文数字**：`"第一期"` 的 `"一"` 不是 `\d`。
6. **引文必须核实**：不能自由发挥，必须是可验证的原文+出处。
7. **macOS provenance** 锁定的文件无法操作，创建新文件代替。

## 推送渠道迁移：Server酱 → Lark

**时间：** 2026-06-25

**原因：** 希望将每周推送从微信迁移到 Lark 群聊。

**实现：**
1. 在 Lark 桌面端创建自定义机器人，获取 Webhook URL（`open.larksuite.com`）
2. 开启签名校验，密钥存入 GitHub Secret `LARK_SIGNING_SECRET`
3. 将 workflow 中的 `curl` 从 Server酱 API 替换为 Lark Webhook
4. 消息格式改为 Lark 交互卡片（`msg_type: "interactive"`），含蓝色标题栏、模型名称、副标题、阅读按钮
5. 签名算法：HMAC-SHA256(key=`timestamp\nsecret`, data="") → Base64
6. Shell 中的 `openssl` 无法正确处理换行符作为 HMAC key，改用 Python 一行脚本计算签名

**Webhook URL 格式：** `https://open.larksuite.com/open-apis/bot/v2/hook/<id>`

**教训：**
- Lark 自定义机器人**必须在桌面端创建**，网页版和移动端无此入口
- 签名算法中 `"\n"` 是 HMAC key 的一部分，bash 的 `echo -ne` 容易出错，Python 更可靠
- 交互卡片的 `note` 元素格式易出错，测试时先用简单卡片逐步验证
