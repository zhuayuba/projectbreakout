# AGENTS.md — 他山之石（破茧计划系列）

## 项目概述

他山之石是破茧计划的一套基于查理·芒格"格栅理论"的跨学科思维模型学习系统。每期围绕一个硬科学核心模型，用人话讲清楚概念，映射到人文社科领域，并布置反向思考题。目标是快速建立跨学科格栅，触类旁通。

项目以**纯静态、自包含的 HTML 文件**交付，无需任何构建工具或服务端——双击 `.html` 文件即可在浏览器中使用。

## 技术约束（极其重要）

### 1. 完全自包含
- 每个 `.html` 文件包含 **CSS + HTML + JS**，不依赖任何外部资源
- 无 npm、无 CDN、无外部样式表、无外部脚本
- 不依赖 Node.js、Python 或任何构建系统

### 2. 纯 ES5 JavaScript（硬性约束）
- **只能使用 `var` 和 `function`**，严禁 `const`、`let`、`=>`、模板字符串 `` `${}` ``
- 所有字符串使用双引号 `"`
- JS 代码包裹在 IIFE 中：`(function(){"use strict"; ... })();`
- **原因**：浏览器在 `file://` 协议下对 ES6+ 语法的解析行为不可预测，使用 ES5 消除所有静默失败的可能

### 3. file:// 协议兼容
- 项目通过 `file://` 协议运行（双击 HTML 文件打开），不经过 HTTP 服务器
- `localStorage` 在 `file://` 下可能不可用，数据存储采用**内存 + localStorage 双层策略**
- 内存数组 `_h` 为主存储（同页面内始终可用），localStorage key `munger_backup` 为辅助持久化

## 每期内容模板

每期 HTML 文件遵循固定结构：

1. **硬科学模型** (🔬) — 讲清楚模型本身，用人话，有生动的日常类比
2. **明珠概念** (💎) — 一句话总结，放在 `.highlight-box` 中
3. **晚间引子** (🌙) — 激发好奇心的问题/场景，串联硬科学和人文现象
4. **人文社科对照** (🏛️) — 2-3 个可折叠 `<details>` 面板，用模型重新解读历史/社会/心理现象
5. **反过来想** (🔄) — 逆向思考题（芒格核心方法论）
6. **启示** (🧠) — 对个人生活/决策的具体启发

### HTML 结构

```
header.hero          ← 主题色 hero banner
  .hero-badge        ← 必须包含阿拉伯数字 "他山之石 · 第X期"
div.container
  div.intro          ← 引言卡片
  section            ← 主体内容
    h3               ← 章节标题

```

## JS 内核规范

JS 内核在每个期文件中约 12KB，包含以下功能模块：

- **右键触发工具栏** — 选中文本 → 右键 → 弹出"标记"/"记笔记"按钮
- **高亮标记** — 用 `<mark class="mh">` 包裹选中文本
- **笔记弹出窗口** — 模态窗口，文本区域输入笔记
- **侧边栏笔记本** — 右下角"📒 笔记本"浮动按钮 → 右侧滑出 380px 侧边栏
- **气泡弹出窗口** — 点击高亮标记显示笔记内容和操作按钮
- **Toast 通知** — 轻量级提示

### 关键变量和 class

- 数据数组：`_h`（内存主存储）
- 高亮元素 class：`mh`
- 侧边栏卡片 class：`mc`
- 浮动按钮 id：`fab`
- 标记计数器 id：`bdg`
- localStorage key：`munger_backup`

### 关键函数

- `info()` — 解析 `.hero-badge` 中的期号（同时支持阿拉伯数字和中文数字）
- `addHl()` — 添加高亮标记
- `delHl()` — 删除高亮标记
- `updHl()` — 更新高亮笔记
- `act()` — 工具栏动作分发
- `mkUI()` — 创建 UI 组件（工具栏、弹出窗口等）
- `mkSide()` — 创建侧边栏
- `renderSB()` — 渲染侧边栏内容
- `restore()` — 从 localStorage 恢复数据
- `toast()` — 显示 toast 通知

## 创建新期次的步骤

1. 确定硬科学模型和期号（如 `06`）
2. **以 `01.html` 的结构为模板**创建 `XX.html`（不要从前一期复制）
3. 修改 CSS hero 色彩主题（每期可用不同颜色）
4. 填入内容，严格遵循固定模板结构
5. **完整复制 JS 内核**到 `</body>` 前，一字不改
6. 确保 `.hero-badge` 内是阿拉伯数字："他山之石 · 第X期"
7. JS 语法检查：用 `node -c` 检查提取出的 JS 是否合法
8. 更新 `破茧计划.html` 首页卡片
9. 更新各期底部的上下篇导航链接

## 已知问题

1. **`index.html` 权限锁定**：被 macOS `com.apple.provenance` 扩展属性锁定，无法操作。使用 `破茧计划.html` 代替。
2. **localStorage 不可用**：`file://` 协议下可能被浏览器禁用。内存存储保证同页面内功能正常。
3. **跨页面数据共享**：通过共享 localStorage key `munger_backup` 实现。同一浏览器中先后打开多期，侧边栏会显示所有期的标记。
4. **右键触发是唯一可靠方案**：`mouseup`/`selectionchange` 事件在 `file://` 下有不可解决的时序冲突。

## 调试历史关键教训

1. **`.hero-badge` 必须用阿拉伯数字**："第一期" 的 "一" 不会被正则 `/\d+/` 匹配到，导致 `info().n` 返回 0，标记数据无法入库。
2. **file:// 下 ES6 静默失败**：`const`/`let`/箭头函数在 `file://` 下脚本可能在加载阶段就崩溃，无任何报错。必须用 ES5。
3. **右键 > 自动弹出**：确定性用户意图事件 > 时序敏感的自动检测。
4. **引文必须逐条核实**：不能凭记忆引用，必须找到原文出处。
