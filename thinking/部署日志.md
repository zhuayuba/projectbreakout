# 思维模式篇 · 部署日志

## GitHub Pages 部署问题记录

### 问题 1：Git 推送失败（`failed to push some refs`）

**原因**：创建 GitHub 仓库时勾选了 "Add a README file"，导致远程仓库有一个 README.md，和本地代码产生冲突。

**解决**：
1. 删除 GitHub 上的仓库（Settings → Danger Zone → Delete repository）
2. 重新创建同名仓库，**三个初始化选项全部不勾**
3. 本地设置 git 身份后提交推送：
   ```bash
   git config --global user.name "zhuayuba"
   git config --global user.email "你的邮箱"
   git commit -m "思维模式篇上线"
   git push -u origin main
   ```

### 问题 2：GitHub Pages 部署超时（`Timeout reached, aborting!`）

**现象**：Actions 里 `pages build and deployment` 显示 `deployment_queued`，然后超时红叉。

**原因**：GitHub Pages 部署队列偶尔会卡住，尤其新仓库首次部署时。

**尝试过但未解决的方法**：
- 添加 `.nojekyll` 文件（无效）
- Re-run failed jobs（无效，仍然排队超时）

**最终解决方法**：
1. 进入仓库 Settings → Pages
2. Branch 先改为 **None** → 点 Save
3. 再改回 **main / (root)** → 点 Save
4. 这个操作触发了新的部署任务，第三次成功（绿勾）

**结论**：遇到部署队列卡住时，重置 Branch 选择比等待重试更有效。
